342 /***************************************************************************//**
343  * @brief
344  * Compute one byte of the CRC.
345  *
346  * @note
347  * 7.6.4.7 CHKSUM Field
348  * The ATA6870N provides the possibility of verifying the transmitted data using a checksum. Setting chksum_ena bit
349  * to 1 in the Ctrl register [default = 0) activates the checksum feature.
350  * The chksum field is an 8-bit checksum computed from the proceeding data [control and data fields, byte 3 to byte n-1). It is
351  * based on the polynomial x8+x2+x1+1. The way it is computed is depicted below:
352 
353  * The checksum is calculated from the CONTROL field and DATA field by a polynomial division. The DATA field can consist of
354  * 1 byte up to 14 bytes [112-bit read-only burst access). The IDENTIFICATION field [2 bytes) is not used to generate the
355  * checksum. The checksum is always sent by the microcontroller, independent of read write mode.
356  * The checksum is in the LFSR [linear feedback shift register) when the complete bitstream [the whole fields of the
357  * transaction) followed by 0x00 have been shifted in the LFSR.
358  * The checksum verification on the complete data transmission was OK when the complete bitstream followed by the
359  * checksum have been shifted in the LFSR, and the content of the LFSR is 0x00. If this is not the case, the receiving
360  * ATA6870N will set the chkError bit in the status register. This will cause an interrupt to the MCU if the chkError is not masked
361  * by the chkErrorMsk bit in the IrqMask register. See the example below. The checksum is serially computed from the 8-bit
362  * value 0x57. So the bitstream 0x5700 is shifted in the LFSR. The resulting checksum is [f6o, f6i, f5i f0i] at the last shift in
363  * cycle:
364  * During an SPI write access, the checksum is computed by the MCU and sent MSB first in the CHKSUM field. For an SPI
365  * read access, the checksum is computed by the ATA6870N and is checked by the MCU. During CHKSUM, MCU has
366  * to send 0x00 on MOSI, and must check that its own LFSR equals 0x00 at the end of CHKSUM field.
367  *
368  * @param[in] u8Data The new data value to add to the CRC.
369  * @param[in] u8InitialCRC The initial CRC conditions.
370  * @return The new CRC
371  * @st_funcMD5 BF1E1427911C29BED3835813AB683097
372  * @st_funcID LCCM650R0.FILE.005.FUNC.005
373  */
374 Luint8 u8ATA6870_LOWLEVEL__CRC(Luint8 u8InitialCRC, Luint8 u8Data)
375 {

382

385  {

404  }
